# Notion AI 스마트 노트 및 배치 지식 요약 도구 🤖📝

🌐 **언어 전환 (Languages)**: [繁體中文](README.md) | [简体中文](README_ZH.md) | [English](README_EN.md) | [日本語](README_JA.md) | **한국어**

이 프로젝트는 **자연어 대화 (Interactive Agent)** 와 **대규모 배치 증분 처리 (Batch Processor)** 를 결합한 Notion 지식 관리 솔루션을 제공합니다.

> 💡 **프로젝트 출처 표기**:
> 본 프로젝트의 대화형 Agent 모듈 실행 및 일부 워크플로우 설계는 Shubhamsaboo의 오픈 소스 프로젝트인 [awesome-llm-apps/notion_mcp_agent](https://github.com/Shubhamsaboo/awesome-llm-apps/tree/main/notion_mcp_agent)에서 영감을 얻어 제작되었습니다. 본 프로젝트는 이 프레임워크를 기반으로 대규모 배치 처리 및 하이브리드 RAG 지식 요약 엔진을 추가 및 확장하여 고도화된 커스터마이징을 제공합니다.

## 🌟 핵심 기능

### 1. 대용량 노트 자동 요약 및 AI 자동 태깅 (Batch Processor)
수천 개의 웹 기사, 전자책, 동영상 노트가 포함된 Notion 소스 데이터베이스를 대상으로 증분 배치 처리를 진행합니다:
* **중복 방지 증분 처리**: 처리되지 않은 페이지(`AI已處理` 체크 박스가 해제된 항목)만 자동으로 추출하고, 완료 후 체크를 표시하여 Token 낭비 및 데이터 중복 입력을 방지합니다.
* **속성 구조화 분할 저장**:
  * **'한 줄 요약'**: Notion의 **`AI摘要`** 속성에 자동 저장.
  * **'3가지 핵심 나리지 포인트'**: Notion의 **`AI知識點`** 속성에 자동 저장.
* **빈 URL 페이지 자동 스크래핑**: URL만 존재하고 본문이 비어 있는 카드(예: 페이스북 공유 링크 등)를 자동 감지하여 본문을 스크래핑하고 제목을 재구성한 뒤 AI 요약 및 태깅을 실행합니다.

### 2. 프로덕션 급 지식 베이스 요약 및 요약 기사 생성 (Knowledge Synthesizer)
요약 및 태깅이 완료된 Notion 데이터베이스에서 여러 기사를 관통하는 테마를 자동으로 분류 및 통합합니다:
* **태그 클렌징 및 중복 제거**: 태그의 대소문자, 공백, 중복을 정리하여 의미를 정렬하고, 너무 광범위한 카테고리(예: `AI工具`, `AI應用`, `自動化` 등)를 필터링하여 테마를 고도로 좁혀나갑니다.
* **'깊은 영감을 준'精選 기사 우선순위**: `深受啟發`(영감을 준) 속성이 활성화된 정선된 기사를 자동 감지합니다. **이 기사가 포함된 테마를 우선적으로 선택**하고, 요약 생성 시 **해당 기사의 전체 텍스트를 RAG의 핵심 콘텍스트로 우선 로드**합니다.
* **최적화된 Markdown 레이아웃**:
  * **다중 헤더 수준 분석**: H1~H6 헤더를 동적으로 처리합니다. 해시 기호(`#`)를 자동으로 제거하고 H4~H6见见见見見见見见見见見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見見见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见见지 못하는 헤더를 Notion이 지원하는 H3(`heading_3`)로 안전하게 매핑하여 정상적인 글꼴 크기로 표시합니다.

## 📖 설정 가이드 (Step-by-Step)

Notion 데이터베이스 및 API 키 설정 단계는 다음과 같습니다:

### 1. Notion 인테그레이션 토큰 획득
1. [Notion Developers 인테그레이션 관리 화면](https://www.notion.so/profile/integrations)에 접속합니다.
2. 우측 상단의 **"+ New integration"**을 클릭합니다.
3. 워크스페이스를 선택하고 이름을 `Notion AI Curation Assistant`로 설정한 후 **"Submit"**을 클릭합니다.
4. 생성된 **Secrets** 아래의 **Internal Integration Token**을 복사합니다. 이것이 `NOTION_API_KEY`입니다.
5. **"인테그레이션 기능 (Capabilities)"**에서 다음 권한이 켜져 있는지 확인합니다:
   * `콘텐츠 읽기`
   * `콘텐츠 업데이트`
   * `콘텐츠 삽입`

### 2. Notion 데이터베이스 ID 획득
1. 대상 Notion 데이터베이스를 브라우저에서 엽니다.
2. 우측 상단의 **"링크 복사"**를 클릭합니다.
3. 워크스페이스 이름을 나타내는 슬래시 `/` 뒤, 물음표 `?` 앞에 있는 32자리 영숫자가 데이터베이스 ID입니다. 예: `981c349065cd4f92bb0ba358b9e0a56d`.

### 3. Notion에서 커넥션 추가
1. 브라우저에서 대상 Notion 데이터베이스를 엽니다.
2. 우측 상단의 **`...`** (점 3개) 아이콘을 클릭합니다.
3. 하단에 있는 **"커넥션 추가"**를 선택합니다.
4. 생성한 인테그레이션(예: `Notion AI Curation Assistant`)을 검색하여 선택하고 **"확인"**을 클릭합니다.

### 4. Google Gemini API 키 획득
* [Google AI Studio](https://aistudio.google.com/)에 접속하여 로그인한 후 **"Get API key"** -> **"Create API key in a new project"**를 클릭하여 생성합니다.

### 5. Agent Platform (구 Vertex AI) 서비스 계정 JSON 인증 정보 설정
GCP 서비스 계정을 통한 프로덕션 용도의 초고속 호출을 원할 경우:
1. [Google Cloud Console](https://console.cloud.google.com/)에 로그인합니다.
2. **"IAM 및 관리"** -> **"서비스 계정"**으로 이동하여 계정을 새로 만듭니다.
3. IAM 역할에서 **`Vertex AI User`** 역할을 검색하여 할당합니다.
4. 서비스 계정 옵션 메뉴에서 **"키 관리"** -> **"키 추가"** -> **"새 키 만들기"**를 선택하고 **JSON** 형식을 다운로드합니다.
5. 다운로드한 파일을 **`gcp-key.json`**으로 이름 변경하여 `Notion_agent/` 루트 디렉터리에 저장합니다(이미 `.gitignore`에 등록되어 있습니다).
6. API 전환: `.env`에서 `USE_VERTEX_AI=false`로 설정하면 일반 API 키를 통한 실행으로 폴백할 수 있습니다.

## 📥 설치 및 환경 구성

### 1. 패키지 설치
```powershell
pip install -r requirements.txt
```

### 2. 환경 변수 설정
`.env.example`을 `.env`로 복사하고 다음을 작성합니다:
```ini
NOTION_API_KEY=ntn_your_notion_token_here
GEMINI_API_KEY=your_gemini_api_key_here
NOTION_DATABASE_ID=your_database_id_here
```
